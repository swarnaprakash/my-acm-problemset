Problem Name : Balancing Symbols

Time Limit : 2s

Many compilers spill out a hundred lines of errors when a symbol is missed in our codes.To avoid this we write a tool that removes all symbols from the code,except parenthesis,brackets and braces and then checks if they are balanced.Your task is to write a program that checks if a given sequence of symbols is balanced or not.For instance [()] is legal but [(]) is wrong.

Input:

Number of test cases t &le 250 on the first line.Each test consists of the sequence which is a string of characters without spaces (length &le 100).The string will contain only the characters '(' , ')' , '{' , '}' , '[' , ']' .

Output:

For each test output "RIGHT" if the sequence of symbols is matched else "WRONG" on a single line.

Sample Input:

5
()
[()]
[(])
[]{}()
([{

Sample Output:

RIGHT
RIGHT
WRONG
RIGHT
WRONG
